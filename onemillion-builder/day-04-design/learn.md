## Learning Frame

- **Mental model:** Design The Product turns the current pipeline artifact into the next artifact or verified product state.
- **What can go wrong:** the harness jumps into tasks without teaching the concept, or the learner lets the agent make product decisions without review.
- **What to ignore today:** ignore polish outside this day's gate. Finish the current stage before expanding scope.

## What You Learn

- turn refined PRD into screens and flows
- learn product design basics
- define MUI design language
- design loading, empty, error, and success states
- review the design before code

## Core Concepts

- Design is product behavior made visible.
- Every important flow needs loading, empty, error, success, partial, and full states.
- MUI gives the course a shared design language so the build agent does not invent random UI.
