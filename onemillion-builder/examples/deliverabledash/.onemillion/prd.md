# DeliverableDash PRD

## 1. Problem

Freelance designers with multiple active clients lose track of deliverables and spend too much time writing weekly status updates.

## 2. User

Primary user: freelance UX or brand designer with 3-8 active clients.

## 3. Core Features

1. **Deliverable tracker** — create, view, update, and delete deliverables with client, due date, and status.
2. **Client dashboard** — filter deliverables by client and status.
3. **AI weekly update draft** — generate a client-ready status update from active deliverables.

## 4. Definition Of Done

- User can sign up, log in, and only see their own deliverables.
- User can complete full CRUD on deliverables.
- User can generate an AI update from selected client deliverables.
- App is deployed and monitored.

## 5. Out Of Scope

- Team accounts
- Calendar sync
- Email sending
- Invoices
- Mobile app
