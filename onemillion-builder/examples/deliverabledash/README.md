# DeliverableDash Example

This is a golden-path example of the `.onemillion/` artifacts a builder creates during the course.

It includes a lightweight sample app mock plus the reference artifacts for what "specific enough" looks like:

- [Sample finished app mock](app/README.md)
- [Course artifacts](.onemillion/project.json)

- Day 1: project shape
- Day 2: problem notes
- Day 3: PRD
- Day 7: AI feature spec
- Day 12: AI acceptance criteria
- Day 13: launch audit
- Day 17: feedback
- Day 18: demo submission

Use it when a verifier says your artifact is too vague.
