# Authoritative Sources Reference

Use this skill when research requires verified, reputable sources.

## Source Tier System

| Tier | Type | Trust Level |
|------|------|-------------|
| 1 | Primary research, official docs, peer-reviewed papers | Highest |
| 2 | Industry reports (Gartner, McKinsey, CB Insights), established news | High |
| 3 | Expert blogs, technical documentation, curated aggregators | Medium |
| 4 | Forums, social media, unverified claims | Lowest — corroborate only |

## By Domain

### Technology
- Official documentation (docs.python.org, nextjs.org, fastapi.tiangolo.com)
- GitHub repositories — check stars, last commit, contributor count
- npm/PyPI package pages — weekly downloads, maintenance status

### Security
- CVE database: cve.mitre.org
- OWASP: owasp.org
- NIST: nvd.nist.gov

### Market Research
- Statista, CB Insights, PitchBook (paid — cite if available)
- Crunchbase (free tier for basic data)
- SimilarWeb (traffic estimates)
- Google Trends (search interest over time — free)

### Business / Legal
- SEC EDGAR (public company filings)
- USPTO (trademark search: tmsearch.uspto.gov)
- Company official press releases > news coverage

## Cross-Referencing Rule

Any factual claim that affects a product or business decision needs 2+ independent Tier 1-2 sources before presenting as fact. If only 1 source found, label it: "[Single source — verify independently]"

## Freshness Rules

| Data type | Max age |
|-----------|---------|
| Market size / TAM | 2 years |
| Competitor pricing | 6 months |
| Technology versions | Check current |
| Security vulnerabilities | Real-time |
