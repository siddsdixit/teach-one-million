# Web App PRD Template

Use this template when `product_type` is `web_app`. Fill every section.

## Template Structure

```markdown
# Product Requirements Document
**App Name:** [Name]
**Product Type:** Web App
**Build Scope:** MVP | Full Product
**Primary Device:** Mobile | Desktop
**Date:** [Date]

---

## Executive Summary

**One-line description:** [What it does in one sentence]

**Product type:** Web App — browser-based application

**Build scope:** [MVP: core workflow only | Full: complete vision]

**Timeline:** [N] weeks

**Primary device:** [Mobile/Desktop]

**MVP Hypothesis:** The core hypothesis can be validated by shipping [FR-X, FR-X]. All other features are staged post-MVP.

---

## Persona

[Full narrative paragraph — role, daily frustration with specifics, primary goal. End with JTBD: "Help [Name] [verb] [outcome] so they can [benefit]."]

---

## Problem Statement

[2-3 paragraphs describing the problem with evidence of pain. Include: who has the problem, how they currently solve it, why current solutions fail.]

---

## Features

| ID | Feature | Description | Priority |
|----|---------|-------------|----------|
| FR-01 | [Name] | [Description] | [MVP] |
| FR-02 | [Name] | [Description] | [POST-MVP] |

---

## Market Research

### Competitors
| Competitor | Strengths | Gaps |
|-----------|-----------|------|

### Market Size
- **TAM:** [Total Addressable Market with assumption]
- **SAM:** [Serviceable Addressable Market]
- **SOM:** [Serviceable Obtainable Market]

### UVP
"[Unique Value Proposition — one sentence]"

### Go-to-Market Channels
1. [Channel 1]
2. [Channel 2]
3. [Channel 3]
```

## Rules for the Idea Agent

- Never mention technologies, frameworks, or hosting in the PRD.
- Persona must be a narrative paragraph, not bullet points.
- Include JTBD sentence: "Help [Name] [verb] [outcome] so they can [benefit]."
- Every feature the builder mentions must appear. [POST-MVP] is sequencing, not cutting.
- Generate a short, memorable app name (1-2 words).
